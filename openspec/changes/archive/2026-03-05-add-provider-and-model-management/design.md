# Provider 管理和 Model 列表设计文档

## Context

当前 Dashboard 已完成基础认证和首页功能。需要添加 Provider 管理功能供管理员使用，以及 Model 列表功能供所有用户查看可用的模型。

**技术约束**：
- 遵循现有的设计规范（`frontend/design/`）
- 使用 Ant Design Vue 组件
- 基于角色控制访问权限

**利益相关者**：
- 管理员：需要管理 Provider 配置
- 普通用户：需要查看可用的模型列表

## Goals / Non-Goals

### Goals
- 提供完整的 Provider CRUD 功能（管理员专用）
- 提供模型列表查看功能（所有用户）
- 添加侧边栏导航

### Non-Goals
- 不实现模型测试功能（后续阶段）
- 不实现模型详情页面
- 不实现 Provider 配置导入/导出

## Decisions

### 1. 布局变更

**决策**：从顶部导航栏改为侧边栏 + 顶部导航栏布局

**理由**：
- 随着页面增多，顶部导航栏空间不足
- 侧边栏更适合展示多层级导航
- 符合常见 SaaS 后台设计模式

**布局结构**：
```
┌─────────────────────────────────────────────────────────────┐
│ Topbar (60px)                                               │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│ Sidebar  │ Main Content                                     │
│ (240px)  │                                                  │
│          │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

### 2. 权限控制

**决策**：基于用户角色控制页面访问

**实现**：
- 路由守卫检查 `userRole`
- 管理员可访问所有页面
- 普通用户只能访问部分页面

### 3. Provider 管理功能

**决策**：提供完整的 CRUD 操作

**功能列表**：
- 列表展示（表格）
- 创建 Provider（表单模态框）
- 编辑 Provider（表单模态框）
- 删除 Provider（确认对话框）
- 启用/禁用 Provider（快捷按钮）
- 重载 Provider（快捷按钮）
- 查看模型列表（抽屉）

### 4. Model 列表展示

**决策**：按 Provider 分组展示所有模型

**展示方式**：
- 折叠面板（Collapse）按 Provider 分组
- 每个模型显示：
  - 模型名称
  - 完整标识（provider/model）
  - 状态标签

## Risks / Trade-offs

### 风险 1：权限判断在前端不安全
**风险**：前端权限控制可以被绕过

**缓解措施**：
- 后端 API 已实现权限控制
- 前端权限控制只为用户体验，不作为安全依赖

### 风险 2：Provider 配置复杂
**风险**：Provider 配置项较多，表单可能复杂

**缓解措施**：
- 使用分步表单或折叠面板
- 添加配置说明和示例
- 提供常用 Provider 的预设模板

2. **是否需要 Provider 状态轮询？**
   - 当前方案不实现，用户手动刷新

3. **Model 列表是否需要搜索功能？**
