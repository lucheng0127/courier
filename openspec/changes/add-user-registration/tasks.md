# 任务清单：用户注册功能

## 任务概述

实现用户自主注册功能，移除管理员创建用户的能力，确保与现有认证体系兼容。

## 实现任务

### 1. 更新 role-based-access 规范

**描述**：修改现有的角色权限规范，移除管理员创建用户的场景。

**文件**：`openspec/specs/role-based-access/spec.md`

**变更**：
- 移除 `### Scenario: 创建用户` 场景
- 移除 `### Requirement: 用户管理权限` 中关于创建用户的内容
- 保留其他用户管理功能（查看、更新、删除、状态管理）

**验证**：确保规范中没有管理员创建用户的描述

---

### 2. 添加注册请求和响应模型

**描述**：在 `model/user.go` 中添加注册相关的数据结构。

**文件**：`internal/model/user.go`

**变更**：
- 添加 `RegisterRequest` 结构体（name, email, password）
- 添加 `RegisterResponse` 结构体（id, name, email, role, status, created_at）
- 标记 `CreateUserRequest` 为废弃（添加 `// Deprecated: 使用用户自主注册替代` 注释）

**验证**：模型字段与规范一致

---

### 3. 实现注册服务方法

**描述**：在 `AuthService` 中添加用户注册方法。

**文件**：`internal/service/auth.go`

**变更**：
- 添加 `Register(ctx, req) (*model.User, error)` 方法
- 实现邮箱唯一性检查
- 实现密码强度验证（至少 8 个字符）
- 使用 bcrypt 对密码进行哈希（cost factor 12）
- 创建用户时设置默认角色为 "user"，默认状态为 "active"

**验证**：
- 注册成功返回用户信息（不含密码）
- 邮箱已存在时返回明确错误
- 密码不符合要求时返回明确错误

---

### 4. 添加注册速率限制中间件

**描述**：创建针对注册接口的速率限制中间件。

**文件**：新建 `internal/middleware/rate_limit.go`

**变更**：
- 实现 `RegisterRateLimit()` 中间件
- 使用内存存储跟踪每个 IP 的注册次数
- 同一 IP 每小时最多 5 次注册请求
- 超过限制返回 429 状态码

**验证**：
- 正常请求不受影响
- 超过限制后返回正确的错误响应

---

### 5. 实现注册控制器方法

**描述**：在 `AuthController` 中添加注册接口处理方法。

**文件**：`internal/controller/auth.go`

**变更**：
- 添加 `Register(ctx)` 方法
- 请求体绑定和验证
- 调用 `AuthService.Register()`
- 处理各种错误情况并返回合适的 HTTP 状态码

**验证**：
- 成功注册返回 201 状态码
- 邮箱冲突返回 409
- 参数错误返回 400
- 速率限制返回 429

---

### 6. 注册路由配置

**描述**：在 `AuthController` 中注册注册接口路由。

**文件**：`internal/controller/auth.go`

**变更**：
- 修改 `RegisterRoutes()` 方法
- 添加 `POST /register` 路由
- 应用 `RegisterRateLimit()` 中间件
- 确保该路由不需要 JWT 鉴权

**验证**：路由在无需鉴权的情况下可访问

---

### 7. 移除管理员创建用户功能

**描述**：从 `UserController` 中移除创建用户的方法。

**文件**：`internal/controller/user.go`

**变更**：
- 移除 `CreateUser()` 方法
- 从 `RegisterRoutes()` 中移除 `users.POST("", c.CreateUser)` 路由

**验证**：
- `POST /api/v1/users` 接口不再可用
- 其他用户管理接口保持正常

---

### 8. 添加注册功能单元测试

**描述**：为注册功能编写完整的单元测试。

**文件**：新建 `internal/service/auth_register_test.go`

**测试用例**：
- 成功注册新用户
- 邮箱已存在时注册失败
- 密码过短时注册失败
- 注册后用户可以使用该密码登录
- 验证密码被正确哈希

**验证**：所有测试用例通过，覆盖率 > 80%

---

### 9. 添加注册接口集成测试

**描述**：为注册 API 端点编写集成测试。

**文件**：新建 `internal/controller/auth_register_test.go`

**测试用例**：
- 正常注册流程
- 缺少必填字段
- 邮箱格式错误
- 邮箱已存在
- 密码过短
- 速率限制

**验证**：所有测试用例通过

**描述**：更新 API 文档，描述注册接口。

**变更**：

## 可验证成果

1. 用户可以通过 `POST /api/v1/auth/register` 自主注册
2. 新注册用户默认为 "user" 角色，状态为 "active"
3. 新注册用户可以使用注册邮箱和密码登录
4. 管理员无法通过 API 创建普通用户
5. 注册接口有速率限制保护
6. 所有功能有对应的单元测试和集成测试
